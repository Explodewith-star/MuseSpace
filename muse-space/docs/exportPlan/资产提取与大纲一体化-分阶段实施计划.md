# 资产提取与大纲一体化 — 分阶段实施计划

> 日期：2026-04-28  
> 基于 `exportPlan/` 下三份设计文档，结合 MuseSpace 当前代码现状制定。

---

## 总览

| 阶段 | 目标 | 核心交付 |
|------|------|---------|
| 第1期 | 导入后自动资产提取 | 原著→候选角色/世界观/文风→建议中心审核→导入正式表 |
| 第2期 | 增强大纲 — 分卷结构 + 局部重做 | 大纲升级为 Volume→Chapter 二层；支持单卷重做 |
| 第3期 | 章节计划 + 章节页生成草稿 | Chapter 扩展计划字段；章节页直接触发草稿生成 |
| 第4期 | 守护增强 + 体验打磨 | 一致性结果结构化展示；文风审查；大纲独立菜单 |

---

## 第1期：导入后自动资产提取

### 目标

原著导入完成（Indexed）后自动触发 3 个提取 Agent，生成候选资产写入 AgentSuggestion，用户在建议中心审核后一键导入正式业务表。

### 后端改动

1. **新增 2 个 AgentDefinition**
   - `WorldRuleExtractionAgentDefinition` — 从原著片段提取世界观规则
   - `StyleProfileExtractionAgentDefinition` — 从原著片段提取文风画像
   - （`CharacterExtractAgentDefinition` 已存在，改造为批量提取模式）

2. **新增 `ExtractNovelAssetsJob`**
   - 从 NovelChunk 均匀采样 10~15 段（覆盖首中尾）
   - 串行调用 3 个 Agent
   - 每个 Agent 输出写入 AgentSuggestion（category 分别为 Character / WorldRule / StyleProfile）
   - 通过 AgentProgressNotifier 推送进度

3. **`EmbedNovelJob` 链式触发**
   - Indexed 成功后 `BackgroundJob.Enqueue<ExtractNovelAssetsJob>(...)`

4. **新增 2 个 SuggestionApplier**
   - `WorldRuleSuggestionApplier` — contentJson → WorldRule 表
   - `StyleProfileSuggestionApplier` — contentJson → StyleProfile 表
   - （`CharacterSuggestionApplier` 已存在可复用）

5. **SuggestionCategories 扩展**
   - 新增 `ExtractedCharacter`、`ExtractedWorldRule`、`ExtractedStyleProfile`
   - 与现有手动创建的 Character/WorldRule/StyleProfile 区分来源

### 前端改动

1. **suggestions/utils.ts** — 新增 3 个 category 标签和图标
2. **建议中心卡片** — 提取类建议渲染：角色名/规则名/文风特征预览
3. **提取类详情页** — 可编辑候选字段后再导入

### 验收标准

- [ ] 导入一本 TXT/MD 小说，等待 Indexed 完成
- [ ] 自动出现"正在提取资产"进度通知
- [ ] 建议中心出现候选角色（多条）、候选世界观规则（多条）、候选文风画像（1条）
- [ ] 可逐条查看详情、编辑后接受→应用
- [ ] 应用后，角色/世界规则/文风页面可看到导入结果

## 第2期：增强大纲 — 分卷结构 + 局部重做

### 目标

大纲从"平铺 N 章"升级为"卷 → 章"二层结构，支持单卷重做。

### 后端改动

- `OutlinePlanAgentDefinition` prompt 改为输出 `{ volumes: [...] }` 格式
- `OutlinePlanJob` 解析二层 JSON
- 新增 `POST outline-plan/{suggestionId}/volumes/{volumeNumber}/regenerate`
- `OutlineSuggestionApplier` 导入时按卷展开写入 Chapter 表

### 前端改动

- `OutlineDetail.vue` 重构为左侧卷导航 + 右侧章节编辑
- 支持"重做本卷"按钮
- 大纲规划表单新增"番外"模式

### 验收标准

- [ ] 生成大纲后可见卷级分组
- [ ] 可展开/折叠每卷查看章节
- [ ] 可重做单卷而不影响其他卷
- [ ] 导入后章节数正确

---

## 第3期：章节计划 + 章节页生成草稿

### 目标

打通"大纲 → 章节配置 → 生成草稿"链路。

### 后端改动

- Chapter 扩展 `Conflict`/`EmotionCurve`/`KeyCharacterIds`/`MustIncludePoints`
- 新增 `ChapterPlanGenerationAgent`
- 新增 `POST /chapters/{chapterId}/generate-draft`

### 前端改动

- 章节详情页新增"章节计划"编辑区
- "自动填充计划"按钮 + "生成本章草稿"按钮

### 验收标准

- [ ] 从章节页直接生成草稿
- [ ] 生成时自动注入章节计划配置
- [ ] 不再需要回自由草稿页手动输入

---

## 第4期：守护增强 + 体验打磨

### 后端

- 文风一致性审查 Agent
- 一致性结果展示在章节详情页

### 前端

- 大纲独立菜单入口
- 长期状态面板（活跃线索/即将回收伏笔）

---

## 设计原则

1. 资产提取结果一律走建议层，不直接写正式表
2. 千章级 PlotThread/BookBlueprint 暂缓，先闭环
3. 复用已有 AgentSuggestion 表，不新建 AgentTask 表
4. SceneDraftSkill 短期复用，长期拆为 ChapterDraftSkill
